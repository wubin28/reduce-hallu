# 减少 AI 幻觉 — 页面设计文档

**日期：** 2026-05-25  
**状态：** 已批准

---

## 概述

将 Next.js Hello World 页面替换为「减少 AI 幻觉」工具页面。该工具通过自动为用户输入的提示词添加「减幻前缀/后缀」，帮助用户得到更可信的 AI 回答。

---

## 功能规格

### 核心行为

1. 用户在多行文本框中输入待优化的提示词
2. 点击「查询事实」按钮后：
   - **空输入处理：** 若输入框为空或仅含空白字符，按钮不响应（禁用状态），无任何弹窗或报错
   - **动画进行中重复提交：** 若打字机动画尚未结束，按钮处于禁用状态，防止重复触发
   - 自动在提示词前添加前缀：`你是专家。`
   - 自动在提示词后添加后缀：`请提供主要观点的3个不同出处的网页链接以便我查验。如果你不知道或查不到，就实说，不要编造。`
   - 保留用户原始输入的换行和空格
   - 清空输入框
   - 以**打字机动画效果**逐字符显示优化后的提示词（在输入框上方的结果区）
3. 结果区右上角有「复制」按钮；点击后文字变为「已复制 ✓」并在 2 秒后恢复，若 Clipboard API 不可用则静默降级（不报错）
4. 「查询事实」是本地纯文本拼接操作，不发起任何网络请求或真实检索，名称仅传达用途意图

## 技术方案

### 文件变更

- `app/page.tsx` — 替换为全新客户端组件（`"use client"`）

### 单元边界

| 单元 | 职责 | 输入 | 输出 |
|------|------|------|------|
| `buildOptimizedPrompt(input)` | 纯函数，拼接减幻提示词 | 原始提示词字符串 | 优化后的完整字符串 |
| `useTypewriter(text, speed)` | 自定义 Hook，打字机动画控制 | 目标文本、字符间隔(ms) | 当前已显示的文本片段 + 是否完成标志 |
| `CopyButton({ text })` | 复制按钮组件，含状态反馈 | 待复制文本 | 点击复制，显示成功/恢复 |
| `HomePage`（page.tsx 默认导出）| 页面顶层，管理输入/输出状态 | — | 渲染整体页面 |

### 页面状态机

| 状态 | 描述 | 输入框 | 结果区 | 按钮 | 复制按钮 |
|------|------|--------|--------|------|----------|
| `empty` | 初始/输入框已清空 | 空，可编辑 | 隐藏 | 禁用 | — |
| `editing` | 有内容输入中 | 有内容，可编辑 | 若有上轮结果则继续显示 | 可点 | 可点（复制上轮完整结果）|
| `typing` | 打字机动画进行中 | 已清空，可编辑 | 显示，逐字出现 | 禁用 | 禁用（结果未完整） |
| `done` | 动画结束 | 可编辑 | 显示完整结果 | 取决于输入框是否有内容 | 可点 |
| `copied` | 点击复制后 2 秒内 | 同上 | 同上 | 同上 | 显示「已复制 ✓」|

**状态转换说明：**
- 用户开始输入新提示词：保持当前结果区不变，显示上一轮结果直到下次点击「查询事实」
- 点击「查询事实」：结果区**替换**为新输出（打字机从头开始），清空输入框，进入 `typing`
- 打字机完成：进入 `done`
- 打字机进行中再次输入：状态为 `typing`，按钮仍禁用

### 按钮禁用条件

- `inputText.trim() === ''`（空输入）
- `isTyping === true`（动画进行中）

### 复制功能

- 复制**完整**目标文本（`outputText`），而非打字机当前已显示片段
- 复制按钮仅在 `done` 状态或 `copied` 状态可用（`typing` 时禁用）
- 实现：`try { await navigator.clipboard.writeText(text) } catch { /* 静默忽略 */ }`
  - 涵盖 API 不可用、权限拒绝、非安全上下文等所有失败情形，均静默降级，不显示任何错误
- 复制成功后按钮文字变为「已复制 ✓」，2 秒后恢复「复制」

### 其他实现细节

- 打字机字符间隔：20ms（固定值，无需配置）
- 响应式：Tailwind `flex-col sm:flex-row` 处理输入区布局

### 减幻提示词模板

```
前缀：你是专家。
后缀：请提供主要观点的3个不同出处的网页链接以便我查验。如果你不知道或查不到，就实说，不要编造。
```
